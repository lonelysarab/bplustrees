Bubble Sort

Given a list of ints stored in an array, a bubble sort causes a pass through the array to compare adjacent pairs of keys. Whenever two keys are out of order with respect to each other, the associated objects are interchanged. Notice that after such a pass, we are assured that the list will have the integer that comes last in order in the final array position. That is, the largest integer will "sink" to the bottom of the array, and smaller values will gradually "percolate" to the top.

If one pass through an array of n keys guarantees that the key last in order is in the appropriate position, then completing the second pass using the same logic will guarantee that the second largest value is in the appropriate position. Repeating the process for a total of n - 1 passes eventually ensures that all objects are in their appropriate positions. In general, on the k th pass through the array, n - k comparisons of pairs must be made.

However, if the algorithm ever passes through the sequence without causing an interchange the list is in order and the algorithm no longer needs to continue. Straight bubble sort does not evaluate for this situation, though.

Beta 2

Neu in Version Beta 2:

  • Der Client verwendet nun das Look and Feel des zugrunde liegenden Betriebssystems. Damit sieht es deutlich besser aus unter Windows und fühlt sich unter Linux besser an.
  • Gaigs wurde überarbeitet, damit es unter Windows besser aussieht und bei den meisten nativen Look and Feel Umgebungen besser harmoniert.
  • Die angepasste Abspielkontrolle für Animal passt besser zum nativen Look and Feel
  • Der Installer wurde auf die letzte Fassung von InstallAnywhere Now! aktualisiert.

Behobene Bugs in Beta 2:

  • Der Fehler bei falsch beantworteten True / False Fragen wurde behoben.

Beta 1

Neu in Version Beta 1:

  • Der Client verbindet sich beim Öffnen nicht mehr mit dem Server, sondern fragt erst nach einer Kategorie und dem Servernamen.
  • Das Öffnen der Info-Seite wurde ersetzt durch eine Anleitung, wie man JHAVÉ benutzt.
  • Der Installer zeigt nun die Versionsnummer bei der Installation an.
  • Die Versionsnummer erscheint auch in der About-Anzeige.
  • Die About-Anzeige wurde geändert. Sie umfasst nun auch Informationen über den Autor und andere Beteiligte am Client.

Behobene Bugs in Beta 1:

  • Fragen mit langen Fragetexten werden nun korrekt angezeigt.

Alpha 3

Neu in Version Alpha 3:

  • Die Version wurde von Prototyp 3 in Alpha 3 umbenannt, da sie nun die meisten Features implementiert.
  • Die Connection History wurde in den Debug-Karteireiter verschoben.
  • Algorithmen mit Eingabegeneratoren sollten nun funktionieren. Zudeme kann man zwischen "alten" und "neuen" Daten auswählen.
  • Der Benutzer kann sich nun am Server einloggen, um an Tests teilzunehmen.
  • Das Fragesystem ist vollständig implementiert.
  • Der Installer wurde von InstallAnywhere Now! 4.5 auf InstallAnywhere Now! 5 aktualisiert.
  • Es gibt nun Kommandozeilenparameter für den Debugmodus, die Auswahl einer Kateogie und die Ausgabe einer Hilfe (--help).
  • Viele größere und kleinere interne Anpassungen.

Behobene Bugs in Alpha 3:

  • Viele größere und kleinere interne Fehlerkorrekturen.

Noch fehlend in Alpha 3:

  • Eine grafische Auswahl der Kategorie vor dem Verbindungsaufbau zum Server.
  • Vermutlich noch einiges weitere, wie die Durchführung der Tests bei Quizzen.

Prototyp 2

Neu im Prototyp 2

  • Die Verbindung zum Server erfolgt beim Systemstart. Visualisierungen ohne Eingabegeneratoren können angezeigt werden.
  • Der neue Connection Karteireiter zeigt Informationen über die Kommunikation zwischen Client und Server an.
  • Der neue Statusbalken zur Verbindung zeigt die letzte Transaktion zwischen Client und Server an.
  • Der neue Debug-Karteireiter ist dazu ausgelegt, Ausgaben nach System.out und System.err anzuzeigen.

Behobene Bugs in Prototyp 2

  • Änderungen zur Verbindungskonfiguration sind nun abspeicherbar.

Noch fehlend in Prototyp 2

  • Das Quizsystem fehlt noch vollständig.
  • Das Testatsystem für "benotete" Quizze fehlt entsprechend ebenfalls.
  • Visualisierungen mit Eingabegeneratoren funktionieren noch nicht.

Prototyp 1

  • Erster Entwurfsprototyp ohne Funktionalität. Dient nur der Demonstration der grafischen Benutzerschnittstelle.
